将 Owen 值（1977）引入图像可解释性，利用语义分割构建层次化特征归因，实现"先组间、再组内"的结构化解释

一、核心问题：Shapley 值的结构性盲区

1.1 Shapley 值的隐含假设

自 SHAP（Lundberg & Lee, NeurIPS 2017）将 Shapley 值确立为特征归因的"唯一公平解"以来，几乎所有的 SHAP 变体——Kernel SHAP、Deep SHAP、Tree SHAP——都共享一个根本性假设：所有特征地位平等、相互独立。在这一假设下，Shapley 值按如下公式计算每个特征 i 的贡献：

φᵢ = Σ_{S ⊆ N\{i}}  |S|! (|N|-|S|-1)! / |N|!  ×  [v(S∪{i}) − v(S)]

这个公式的核心含义是：对所有可能的特征排列求边际贡献的期望。注意，这里 S 是任意特征子集，所有特征可以自由组合——没有任何结构或先验分组约束。

1.2 现实中的特征并非"原子化"

然而，现实世界中的特征几乎从不满足这个假设：

领域特征存在天然分组举例
计算机视觉像素 → 语义区域（狗/草地/天空）同一语义区域内的像素高度相关
网络流量TCP 字段 → 端口组 / 标志位组 / 窗口组协议层 → 字段组 → 具体字段
医疗诊断检查指标 → 血液组 / 影像组 / 症状组组内指标相关，跨组独立
自然语言Tokens → 短语 → 从句 → 句子树形层次结构
核心矛盾：Shapley 值将所有特征视为"N 个等价的原子个体"，但真实世界的特征天然具有层次化分组结构。当我们将分组特征强行扁平化，不仅损失了结构信息，还可能导致违反直觉的归因结果——例如，高度相关的像素可能被分配截然不同的重要性分数，因为它们被当作独立个体处理。

二、Owen 值：层次化 Shapley 的自然延伸

2.1 什么是 Owen 值

Owen 值（Owen, 1977）是 Shapley 值在带联盟结构（coalition structure）的合作博弈中的推广。如果 Shapley 值回答的问题是"每个玩家（特征）对总收益的公平分配是多少？"，那么 Owen 值回答的问题是"在玩家已经形成组（联盟）的前提下，组间和组内如何公平分配？"

2.2 关键概念：联盟结构（Coalition Structure）

给定 N 个特征的集合 N = {1, 2, ..., n}，一个联盟结构（分区）P = {G₁, G₂, ..., Gₖ} 将特征划分到 k 个互不相交的组中。Owen 值的计算分两个层次：

第一层：组间排列
对 k 个组的全排列求期望
→
第二层：组内排列
对组内特征的全排列求期望
→
Owen 值
φᵢ = 组贡献 × 组内占比

2.3 数学定义

对于特征 i ∈ Gⱼ（第 j 个组），其 Owen 值定义为：

Owᵢ(v, P) = Σ_{Q ⊆ P\{Gⱼ}}  Σ_{T ⊆ Gⱼ\{i}}
        |Q|! (k-|Q|-1)! / k!
      × |T|! (|Gⱼ|-|T|-1)! / |Gⱼ|!
      × [v(∪_{g∈Q} Gg  ∪  T  ∪  {i}) − v(∪_{g∈Q} Gg  ∪  T)]

这个公式虽然看起来复杂，但其结构与 Shapley 值完全平行——只是将"一次排列"分解为"外层组排列 × 内层特征排列"。如果每个组只有一个特征（k=n, |Gⱼ|=1），Owen 值退化为标准的 Shapley 值。

2.4 Owen vs Shapley：结构化对比

适用场景无先验分组知识的扁平特征具有天然层次结构的特征
效率公理∑ φᵢ = v(N) − v(∅)

T-property（全加性/总贡献性质）：Owen 值独有的核心公理。它确保了层次化解释的加性一致性——分配给一个组的贡献恰好等于该组内所有成员分配到的贡献之和。这一性质使得"组级解释"和"特征级解释"可以无缝衔接、自洽统一。

三、O-Shap 核心设计：语义感知的层次化解释

3.1 核心创新

O-Shap 的核心创新在于将语义分割（Semantic Segmentation）作为"联盟结构"的构造器：

输入图像
→
语义分割
生成语义区域（联盟）
→
Owen 值
组间归因
→
Owen 值
组内归因
→
层次化
解释输出

3.2 方法流程

步骤 1：语义分区（Semantic Partitioning）

使用预训练的语义分割模型（如 DeepLab、SAM）将输入图像分割为语义上有意义的区域。每个区域构成一个"联盟"（coalition）。例如，一张"草地上的狗"图片被分割为：

  • G₁（狗区域）：狗身体的像素集合
  • G₂（草地区域）：草地背景的像素集合
  • G₃（天空区域）：天空的像素集合

步骤 2：组间 Owen 归因

将每个语义区域视为一个"超级特征"（super-feature），计算区域级别的 Owen 值。这回答了"哪个语义区域对模型预测最重要？"

步骤 3：组内 Owen 归因

在每个语义区域内部，将像素视为不可再分的原子特征，计算像素级别的 Owen 值。这回答了"在关键区域内部，哪些像素最具有决定性？"

步骤 4：层次化可视化

将两层归因结果合并为统一的层次化解释——既展示区域级的重要性热力图，又展示像素级的精细归因。

3.3 为什么语义分区优于任意分组

Owen 值本身不规定如何划分联盟——任何分区都可以。但 O-Shap 选择语义分割的理由非常关键：

解释的可理解性：人类以语义概念（"狗"、"草地"）理解图像，而非以像素坐标。语义分区使解释与人类认知对齐。
结构合理性：同一语义区域内的像素在模型表征空间中高度相关，将它们作为一组既符合数据分布也符合模型内部表示。

四、具体示例：从 Shapley 到 O-Shap

4.1 案例设定

考虑一张"草地上奔跑的狗"的图像，输入到图像分类模型（如 ResNet 或 ViT），模型预测为"狗"（置信度 0.95）。我们想解释：哪些像素驱动了这个预测？

4.2 传统 Shapley 解释的问题

如果对所有像素一视同仁地计算 Shapley 值，会得到一张像素级重要性热力图。问题在于：

  • 狗耳朵上的两个相邻像素可能得到非常不同的 Shapley 值（因为 Shapley 将它们视为独立特征，而实际上它们共同构成"耳朵"这一语义概念）
  • 解释结果的碎片化：热力图上的高亮区域可能是碎片状的，缺乏语义级别的聚合
  • 无法回答"整个狗区域"贡献了多少——只能回答"每个像素"的贡献，需要额外聚合

4.3 O-Shap 的层次化解释结果

完整案例：狗在草地上 
【第一层：语义区域级归因】（组间 Owen 值）

│ 语义区域          │ Owen 贡献    │ 解读                  │

这种层次化解释比传统 Shapley 的扁平热力图提供了结构化认知：不仅知道"狗区域最重要"，还知道"眼睛像素是狗区域中最关键的子成分"。这是人类推理的自然层次——先定位"什么对象"，再分析"对象的哪部分"。

五、T-property：层次化解释的数学基石

5.1 T-property 的定义

T-property（Total Contribution Property）是 Owen 值独有的公理，也是它与 Shapley 值的关键区别：

T-property：对于任意联盟结构 P 和任意组 Gⱼ ∈ P，组 Gⱼ 本身的 Owen 值（将整组作为一个玩家参与组间博弈）等于该组所有成员的个体 Owen 值之和。

数学表达：φ(Gⱼ) = Σ_{i∈Gⱼ} φᵢ

5.2 为什么 T-property 至关重要

如果没有 T-property，"层次化解释"就是虚假的——你无法同时声称"狗区域贡献了 70%"和"狗区域的像素贡献之和为 70%"。T-property 保证了这两个陈述的一致性。

对比：如果对 Shapley 值结果进行事后分组求和：

  • Shapley + 事后聚合：先计算所有像素的独立 Shapley 值，再按语义区域求和。这≠ Owen 值，因为 Shapley 值在计算时未考虑组结构，组间交叉项未被正确处理。
  • Owen 值：在计算过程中就嵌入了组结构，组间博弈和组内博弈被显式建模，T-property 自然成立。

5.3 效率公理的层次化推广

Shapley 值的效率公理（Efficiency）要求所有特征贡献之和等于总预测值。Owen 值继承并推广了这一公理：

  • 全局效率：Σ_{所有组} φ(Gⱼ) = v(N) − v(∅)（组级效率）
  • 组内效率：Σ_{i∈Gⱼ} φᵢ = φ(Gⱼ)（T-property，组内效率）
  • 组合：Σ_{所有特征} φᵢ = v(N) − v(∅)（特征级效率，自动满足）

六、网络流量可解释性的 Owen 视角

6.1 网络流量特征的天然层次结构

网络流量数据的特征具有极其自然的层次化分组结构，这使得 Owen 值（而非扁平 Shapley）更适合：

第一层：协议层
├── TCP 流量组
│   ├── 端口子组：源端口、目标端口
│   ├── 标志位子组：SYN、ACK、FIN、RST、PSH
│   ├── 窗口子组：窗口大小、窗口缩放因子
│   └── 选项子组：时间戳、MSS、SACK
│
├── UDP 流量组
│   ├── 端口子组：源端口、目标端口
│   └── 长度子组：UDP 长度、校验和
│
└── 统计特征组
    ├── 时序子组：包间隔、流持续时间
    ├── 载荷子组：载荷熵、字节分布
    └── 计数子组：包数、字节数

6.2 层次化解释的实践价值

在网络安全场景中，层次化解释提供了更具操作性的分析能力：

场景Shapley 能回答的问题Owen 能额外回答的问题
恶意流量检测"哪些特征字段最重要？""是协议头特征组驱动，还是统计特征组驱动？→ 在协议头组内，标志位还是端口号更关键？"
加密流量分类"哪个特征贡献最大？""载荷统计特征组和时序特征组各自的贡献占比是多少？→ 时序组内，包间隔还是流持续时间更关键？"
模型调试"模型在关注什么特征？""模型是在做合理的层次化推理，还是仅依赖某个单一组内的少量特征？"
关键洞察：在流量预训练模型（如 ET-BERT、MM4flow）中，虽然特征被 token 化并输入 Transformer，但 token 对应的原始流量字段仍保留着天然的层次关系。将 O-Shap 的层次化归因思想引入流量模型解释，可以在 Transformer 的"黑盒"输出和原始流量结构之间建立桥梁。

七、计算方案与局限性

7.1 计算挑战

Owen 值的精确计算需要对两层排列（组间排列和组内排列）求期望，其复杂度远高于单层 Shapley。O-Shap 需要同时处理：

  • 外层采样：k 个语义区域的排列空间（k! 种可能）
  • 内层采样：每个区域内 |Gⱼ| 个像素的排列空间（|Gⱼ|! 种可能）
  • 双重组合爆炸：实际计算中需要对两层同时采样

O-Shap 采用蒙特卡洛采样策略来近似 Owen 值，这与 Owen MC Approximation（2023）文中的理论框架相辅相成：后者提供了严格的误差界保证，前者将其应用于语义感知的图像解释场景。

7.2 语义分割质量依赖

O-Shap 的联盟结构完全由语义分割模型提供。这意味着：

关键限制：
  • 如果语义分割出错（例如将狗的一部分错误分割为"草地"），O-Shap 的归因也会继承这个错误
  • 语义分割的粒度直接影响解释的粒度——过粗的分割丢失细节，过细的分割消解了层次化的优势
  • 对于语义分割模型本身无法定义的场景（如无明确对象的抽象纹理），O-Shap 的优势受限
  • 语义分割质量与下游任务的相关性：语义分割的"正确"划分未必是模型推理的"最优"划分

7.4 适用性与边界

  • 最适用的场景：特征具有清晰的语义分组结构，且分组与模型推理逻辑对齐（如图像的语义分割）
  • 较适用的场景：领域知识可提供合理的分组先验（如网络流量的协议层次）
  • 有限的场景：特征无天然分组，或分组与模型推理逻辑无关联（如纯统计特征的随机分组）

八、关键贡献与总结

8.1 O-Shap 的三重贡献

1. 语义感知的层次化解释：首次将语义分割引入 Owen 值框架，使联盟结构具有人类可理解的语义含义，而非任意聚类。
2. 两层归因统一框架：同时提供组级和特征级解释，通过 T-property 保证两层的一致性，实现了"先宏观、后微观"的层次化推理。
3. 跨域推广潜力：虽然 O-Shap 聚焦图像，但其方法论——"领域先验定义联盟 + Owen 值层次归因"——可直接推广到任何具有自然分组结构的领域（网络流量、医疗、NLP 等）。

8.3 本仓库中的定位

O-Shap 在仓库的知识体系中处于Owen 值应用层，与另外两篇 Owen 论文形成三角互补：

  • Owen MC Approximation (2023) → 提供近似计算的理论保障（怎么算得准）
  • Owen Tree Models (2024) → 提供精确计算的闭式解（在特定模型上怎么算得精确）
  • O-Shap (2025) → 提供语义感知的应用框架（怎么用得有意义）

三者共同构成 Owen 值从理论到实践的完整技术栈。
